🎵 Project Overview

This project uses Spotify's audio feature dataset from Kaggle to explore what drives track popularity. We'll analyze seven audio features and train two classifiers — a Support Vector Machine (SVM) and a Decision Tree — to predict whether a track will be popular.

  • Dataset: Spotify Tracks Dataset (Kaggle)
  • Target: Popularity score (0–100)
  • Features: danceability, energy, loudness, tempo, liveness, valence

Step 3 · Preprocessing & Feature Scaling
[6] Markdown

Why Feature Scaling?

SVM is distance-based, meaning features with large ranges (like loudness: –60 to 0 dB or tempo: 60–220 BPM) will dominate over features in 0–1 ranges (danceability, energy). StandardScaler normalizes each feature to mean=0, std=1, ensuring fair comparison.

Step 4 · Model Training & Evaluation
[9] Code Train SVM & Decision Tree classifiers 
# Model 1: Support Vector Machine
clf_svm = svm.SVC(kernel='rbf', C=1.0, random_state=42)
clf_svm.fit(x_train, y_train)
pred_svm = clf_svm.predict(x_test)
score_svm = accuracy_score(y_test, pred_svm)

# Model 2: Decision Tree
clf_dt = tree.DecisionTreeClassifier(max_depth=10, random_state=42)
clf_dt.fit(x_train, y_train)
pred_dt = clf_dt.predict(x_test)
score_dt = accuracy_score(y_test, pred_dt)

💡 Analysis & Next Steps

Both models show low accuracy on exact popularity score prediction — this is expected because popularity spans 0–100 and we're treating it as a multi-class classification problem. The Decision Tree edges out SVM by ~0.9%. A stronger approach would be to bin popularity into tiers (Low / Medium / High), which would reduce class count dramatically and is more meaningful for recommendations. Alternatively, framing this as a regression problem with RMSE as the metric would be more appropriate for a continuous target like popularity.

Step 5 · Conclusions & Future Work
[10] Markdown

📌 Conclusions

This project demonstrates an end-to-end ML workflow on real-world data — from cleaning and EDA through feature engineering and model comparison. Key takeaways:

  • Energy and Loudness are highly correlated (r≈0.76), suggesting potential dimensionality reduction.
  • Popularity is weakly correlated with individual audio features — it's a complex combination, not a single driver.
  • Decision Tree outperforms SVM slightly on this task, likely due to popularity's non-linear boundaries.

🚀 Potential Improvements

  • Bin popularity into 3 tiers (Low / Mid / High) to improve classification accuracy
  • Try Random Forest or XGBoost for better ensemble performance
  • Add genre as a categorical feature (one-hot encoded)
  • Use cross-validation instead of a single train/test split for more robust evaluation
  • Frame as a regression problem using R² and RMSE
