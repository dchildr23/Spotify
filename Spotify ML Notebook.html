🎵 Project Overview — v2

This project uses the Spotify Tracks Dataset from Kaggle (~114,000 tracks) to predict track popularity two ways: as a 3-tier classification problem (Low / Mid / High) and as a regression problem (predicting exact score with R² and RMSE). All models are evaluated using 5-fold cross-validation for reliable, generalizable results.

  • Features: 9 audio features + genre one-hot encoded (top 20 genres)
  • Classifiers: Random Forest, Gradient Boosting, XGBoost — 5-fold StratifiedKFold
  • Regressors: Random Forest, Gradient Boosting, XGBoost — 5-fold KFold
  • Metrics: Accuracy (classification) · R² and RMSE (regression)

Step 4 · Feature Engineering
[6]Markdown

Two key improvements over v1

1. Popularity binning — Instead of treating popularity as 100 classes, we collapse it into 3 meaningful tiers. This makes classification tractable and the accuracy metric actually interpretable.

2. Genre one-hot encoding — The track_genre column is one-hot encoded with the top 20 genres. Genre consistently ends up as one of the most important features — a track being "classical" vs "hip-hop" has a massive effect on expected popularity range.

[7]CodeBin popularity + one-hot encode genre
# Bin popularity into 3 tiers
def bin_popularity(score):
    if score <= 33:  return "Low"
    elif score <= 66: return "Mid"
    else:             return "High"

df["popularity_tier"] = df["popularity"].apply(bin_popularity)

# One-hot encode top 20 genres
TOP_N = 20
top_genres = df["track_genre"].value_counts().nlargest(TOP_N).index
df["track_genre_filtered"] = df["track_genre"].where(df["track_genre"].isin(top_genres), other="Other")
genre_dummies = pd.get_dummies(df["track_genre_filtered"], prefix="genre")

print(f"Popularity tier counts:\n{df['popularity_tier'].value_counts()}")
print(f"\nGenre dummy columns: {genre_dummies.shape[1]}")

The dataset is imbalanced — Low makes up ~51% of tracks. This means a model that always predicts "Low" would score ~51% accuracy without learning anything. Watch for this when interpreting results — we want the model to beat that naive baseline meaningfully.

# Assemble feature matrix (audio + genre dummies)
df_features = pd.concat([df[audio_features], genre_dummies], axis=1)

Step 6 · Classification — Popularity Tier Prediction
[9]Markdown

Why 5-Fold Cross-Validation?

A single train/test split gives you one accuracy number that could be lucky or unlucky depending on which samples ended up in the test set. StratifiedKFold(n_splits=5) splits the data 5 ways, trains and evaluates on each fold, and returns 5 accuracy scores. The mean ± std across folds tells you both performance and stability — a model with high mean but high std is inconsistent and may not generalize.

[10]CodeTrain 3 classifiers with StratifiedKFold CV
cv_clf = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)

classifiers = {
    "Random Forest":     RandomForestClassifier(n_estimators=100, max_depth=15, random_state=42, n_jobs=-1),
    "Gradient Boosting": GradientBoostingClassifier(n_estimators=100, learning_rate=0.1, max_depth=5, random_state=42),
    "XGBoost":           XGBClassifier(n_estimators=100, learning_rate=0.1, max_depth=5, eval_metric="mlogloss", random_state=42, verbosity=0),
}

for name, model in classifiers.items():
    scores = cross_val_score(model, X, y_tier, cv=cv_clf, scoring="accuracy", n_jobs=-1)
    print(f"{name}: {scores.mean():.4f} ± {scores.std():.4f}")

Step 7 · Regression — Exact Popularity Score
[11]CodeTrain 3 regressors with KFold CV — R² and RMSE
cv_reg = KFold(n_splits=5, shuffle=True, random_state=42)

regressors = {
    "Random Forest":     RandomForestRegressor(n_estimators=100, max_depth=15, random_state=42, n_jobs=-1),
    "Gradient Boosting": GradientBoostingRegressor(n_estimators=100, learning_rate=0.1, max_depth=5, random_state=42),
    "XGBoost":           XGBRegressor(n_estimators=100, learning_rate=0.1, max_depth=5, random_state=42, verbosity=0),
}

for name, model in regressors.items():
    r2   = cross_val_score(model, X, y_score, cv=cv_reg, scoring="r2")
    rmse = np.sqrt(-cross_val_score(model, X, y_score, cv=cv_reg, scoring="neg_mean_squared_error"))
    print(f"{name}  R²: {r2.mean():.4f} ± {r2.std():.4f}  |  RMSE: {rmse.mean():.2f} ± {rmse.std():.4f}")

💡 Regression Interpretation

XGBoost leads with R²≈0.40 and RMSE≈17.5 — meaning it explains ~40% of popularity variance and is off by about 17 popularity points on average. This is a honest, reasonable result given that popularity is heavily driven by factors outside the audio data (marketing, artist fame, virality). The consistency across folds (low std) confirms the model generalizes well — it's not overfit to one lucky split.

Danceability and energy, while correlated with each other, have modest individual importance — confirming that no single audio feature alone drives popularity.

📌 Conclusions

This project demonstrates a complete, production-style ML pipeline on real-world data. Key takeaways from v2:

  • Binning popularity into 3 tiers transformed accuracy from ~7% (meaningless) to ~69% (genuinely useful)
  • Genre is the most important feature — adding it via one-hot encoding significantly improved all models
  • XGBoost leads both classification (68.9%) and regression (R²=0.397, RMSE=17.5) with the tightest cross-validation variance
  • 5-fold CV confirms results are stable — low std across folds means models generalize, not just memorize
  • R²≈0.40 is honest: audio features explain ~40% of popularity, the rest is context (marketing, virality) we don't have
